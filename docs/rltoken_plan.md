
# 实验与研究设计报告 (V2)

**项目名称**：基于 RL Token 的 Flow-VLA 残差强化学习微调——从真机到仿真的验证框架

**英文拟定**：Residual RL Fine-Tuning for Flow-based VLAs via RL Token: A Simulation Validation Framework

---

## 一、研究动机与核心定位

### 1.1 现有技术痛点

**VLA 的"最后一毫米"与"最后一步"困境**：以 $\pi_{0.5}$/$\pi_{0.6}$ 为代表的基于流匹配（Flow-matching）的 VLA 经过大规模预训练和监督微调后，在 LIBERO 等标准任务上已能达到 97%+ 的成功率。然而高成功率掩盖了两个关键问题：

- **执行效率低下**：VLA 在接触关键阶段频繁出现"探测行为"——反复接近-回退-重调整，导致任务完成步数远超必要。RL Token 原论文在 Ethernet 任务上观测到 VLA 完成步数为强化学习策略的 2-3 倍。
- **全量强化学习微调不可行**：$\pi_{RL}$/RECAP 等方案需解冻 VLA 主干进行同策略训练（PPO/GRPO），样本效率极低（百万级交互），且存在预训练特征灾难性遗忘风险。

**RL Token 方法的核心优势与待验证问题**：Physical Intelligence 提出的 RL Token 方法在真机部署中展示了显著的吞吐率提升（关键阶段 3 倍加速）。但该方法：

- 仅在 $\pi_{0.6}$（闭源）上验证，未在其他开源 VLA 上测试
- 仅在真机精密操作任务上验证
- 强依赖人类接管和人工稀疏奖励，未验证纯仿真自动化训练的可行性

### 1.2 本研究的核心定位

本项目将 RL Token 方法从真机精密操作迁移至**仿真通用任务环境**，回答以下核心问题：

> **在监督微调基线已经很强（约 97%）的常见操作任务上，RL Token 的残差编辑能否显著提升任务执行效率（吞吐率），同时维持高成功率？**

核心实验设计：冻结 $\pi_{0.5}$ 作为主干网络，基于 rlt-openpi 代码框架实现 RL Token + TD3 演员-评论家网络，在 LIBERO 基准测试上对比**吞吐率**和**成功率**。

## 二、技术架构

### 2.1 系统总览

```
阶段一（离线，RL Token 表征训练）：          阶段二（在线，仿真强化学习训练）：

  π₀.₅（冻结）                               π₀.₅（冻结）
       |                                         |
  VLM 词元嵌入 z_{1:M}                     VLM 词元嵌入 → z_rl
       |                                         |
  [RL Token 编码器] → z_rl                [RL Token 编码器]（冻结）
       |                                         |
  [RL Token 解码器] → L_ro                z_rl + 本体感知 + 参考动作块
                                                  |
                                            [TD3 演员网络] → 精修动作块
                                            [双 Q 评论家网络] → Q 值
                                                  |
                                            LIBERO env.step() + env.check_success()
```

### 2.2 基础 VLA：$\pi_{0.5}$

选择依据：

- $\pi_{0.6}$ 是 $\pi_{0.5}$ 的直接衍生（同架构，仅预训练数据量、VLM 主干、动作专家规模不同），RL Token 的编码器-解码器接口完全兼容
- LeRobot 提供 `lerobot/pi05_base` 和 `lerobot/pi05_libero`（已在 LIBERO 上监督微调的检查点），可直接作为冻结主干
- 第三方实现 rlt-openpi 已基于 OpenPI/$\pi_{0.5}$ 搭建完整阶段一 + 阶段二流水线

### 2.3 RL Token 表征提取（阶段一）

参照 RL Token 原论文和 rlt-openpi 实现：

- 在冻结的 $\pi_{0.5}$ 上添加轻量级编码器-解码器 Transformer
- 编码器将 VLM 最后一层词元嵌入 $z_{1:M}$ 压缩为单个 $z_{rl}$ 向量（256 维）
- 解码器从 $z_{rl}$ 重建 $z_{1:M}$，构建信息瓶颈
- 训练数据：LIBERO 演示数据集（LeRobot 格式，已有）
- 损失函数：$L_{ro} = \mathbb{E}\left[| \text{Decoder}(z_{rl}) - \text{sg}(z_{1:M}) |_2^2 \right]$
- 可选：$\alpha > 0$ 时对 VLA 进行轻度联合微调（$L_{ro} + \alpha \cdot L_{vla}$）

### 2.4 在线强化学习：块级 TD3（阶段二）

冻结 $\pi_{0.5}$ + 编码器，训练轻量演员-评论家网络：

**演员网络**（约 170K 参数，2 层 MLP）：

- 输入：$z_{rl}$ + 本体感知状态 + 参考动作块 $\tilde{a}_{t:t+C-1}$
- 输出：精修动作块 $a_{t:t+C-1}$（残差形式，末层零初始化）
- 参考动作丢弃：训练时以 50% 概率丢弃 $\tilde{a}$ 输入，强制演员网络学会依赖 $z_{rl}$

**评论家网络**（双 Q 网络，TD3）：

- 双 Q 网络，取最小值减少过估计偏差
- 宏动作贝尔曼更新：$y = \sum_{i=0}^{C-1} \gamma^i r_{t+i} + \gamma^C \min_{j=1,2} Q_{\theta_j'}(s_{t+C}, a')$

**行为克隆正则化**：

- $L_{actor} = -Q(s, a) + \beta \cdot | a - \tilde{a} |_2^2$
- $\beta = 0.5$（rlt-openpi 默认值），约束演员网络不偏离 VLA 先验太远

**关键超参数**（参照 rlt-openpi 默认配置）：

|参数|值|说明|
|---|---|---|
|动作块长度 $C$|5-10|宏动作步长|
|$z_{rl}$ 维度|256|信息瓶颈维度|
|演员网络隐层维度|256|2 层 MLP|
|评论家集成|双 Q 网络|减少过估计|
|行为克隆系数 $\beta$|0.5|VLA 先验约束强度|
|参考动作丢弃率|50%|探索与利用平衡|
|更新-数据比|5|每个环境步的梯度更新次数|
|折扣因子 $\gamma$|0.99|长期回报折扣|
|目标网络软更新系数 $\tau$|0.005|Polyak 平均|

### 2.5 真机 → 仿真适配（核心工程改动）

|模块|真机版本（rlt-openpi）|仿真版本（本项目）|
|---|---|---|
|环境接口|`make_franka_env`（Franka+DROID）|`make_libero_env`（LIBERO gym 封装）|
|奖励信号|人类键盘输入（s/f/p）|`env.check_success()` 自动判定，回合结束时稀疏二值奖励|
|人类介入|VR 控制器接管|去掉；用监督微调演示数据预填经验回放缓冲区替代|
|控制频率|50Hz 真机|仿真环境原生频率|
|环境重置|人工重置|`env.reset()` 自动重置|

## 三、评估指标

### 3.1 主要指标：吞吐率

参照 RL Token 原论文的核心评估方式：

- **平均完成步数**：成功回合从开始到 `check_success()=True` 的平均步数
- **吞吐率**：单位步数预算内的成功完成次数
    - 定义：$\text{吞吐率} = \frac{\text{成功回合数}}{\text{总步数预算}} \times \text{归一化常数}$
- **步数分布**：成功回合步数的分布（中位数、P25、P75），可视化对比监督微调与 RL Token

### 3.2 辅助指标：成功率

- 标准成功率：成功回合数 / 总回合数
- 用于确认 RL Token 没有破坏 VLA 原有能力（成功率不应显著下降）

### 3.3 全部实验均同时报告两个指标

包括阶段一的监督微调基线评估——统计 `pi05_libero` 在每个 LIBERO 子任务集上的平均完成步数，作为吞吐率的基准线。

## 四、实验设计与预期结果

**测试环境**：LIBERO 基准测试（Spatial、Object、Goal、Long / Libero-10） **基础 VLA**：`lerobot/pi05_libero`（$\pi_{0.5}$ LIBERO 监督微调版）

### 表 1：监督微调基线与 RL Token 性能对比（预估值）

验证核心主张：RL Token 在维持高成功率的同时显著提升执行效率。

|方法|LIBERO 子集|成功率|平均步数 ↓|吞吐率 ↑|
|---|---|---|---|---|
|$\pi_{0.5}$ 仅监督微调|Spatial|97.0%|~140 步|1.0×（基线）|
|**$\pi_{0.5}$ + RL Token**|**Spatial**|**96.5%**|**~85 步**|**1.6×**|
|$\pi_{0.5}$ 仅监督微调|Object|99.0%|~120 步|1.0×（基线）|
|**$\pi_{0.5}$ + RL Token**|**Object**|**98.0%**|**~80 步**|**1.5×**|
|$\pi_{0.5}$ 仅监督微调|Long / Libero-10|96.0%|~210 步|1.0×（基线）|
|**$\pi_{0.5}$ + RL Token**|**Long / Libero-10**|**95.0%**|**~120 步**|**1.75×**|

_注：监督微调成功率为 LeRobot 官方数据。平均步数为基于任务复杂度的合理估计，实际值以实验为准。RL Token 提升幅度参照原论文真机结果（Ethernet 任务 2 倍步数减少）并做保守折扣（仿真无人类介入辅助）。长序列任务因冗余探测行为更多，RL Token 优化空间更大。_

### 表 2：组件消融实验（预估值）

在 LIBERO-Spatial 上统一评估，验证各组件贡献。

|变体|成功率|平均步数|现象分析|
|---|---|---|---|
|**完整方法**（RL Token + 块级 TD3 + 行为克隆正则 + 参考动作透传）|**96.5%**|**~85**|完整系统，快速收敛，执行流畅|
|去除 RL Token（替换为 ResNet 编码器）|95.0%|~125|缺乏 VLA 内部空间语义，吞吐率提升仅约 10%|
|去除动作块（$C=1$，单步动作预测）|80.0%|~180|稀疏奖励下有效视界过长，评论家梯度传播困难|
|去除行为克隆正则（$\beta=0$）|70.0%|不稳定|最大单因素性能下降，探索过度导致策略崩溃风险|
|去除参考动作透传（去掉 $\tilde{a}$ 输入）|94.0%|~100|最终性能接近完整版但收敛显著变慢|

_注：消融数据参照 RL Token 原论文 Fig. 7-8 的相对趋势估计。行为克隆正则和动作块是两个最关键组件。_

### 表 3：少样本设定（阶段三扩展实验，预估值）

验证 RL Token 在数据不足条件下的补偿能力。

**工程意义**：实际部署中大多数情况下监督微调演示数据的大致轨迹是对的，但局部轨迹不够精细导致成功率上不去。少样本设定模拟了这种"轨迹大致对但局部差一点"的场景，恰好是 RL Token 残差修正最擅长解决的问题。

**实现方式**：使用 `lerobot/pi05_base`（未在 LIBERO 上微调的版本）在不同比例的演示数据上手动监督微调，得到不同强度的基线，再分别嫁接 RL Token。

|演示数据比例|$\pi_{0.5}$ 仅监督微调成功率|+ RL Token 成功率|+ RL Token 平均步数|
|---|---|---|---|
|100%（`pi05_libero`）|97.0%|96.5%（维持）|~85 步（↓39%）|
|20%（手动微调）|~68%|~82%（↑14pp）|~110 步|
|10%（手动微调）|~50%|~65%（↑15pp）|~130 步|

_注：少样本实验工作量较大（需重新监督微调），优先级低于吞吐率主实验。放在阶段三论文准备阶段开展。_

## 五、代码实现路线

### 5.1 代码基座选择

**主要参考**：[rlt-openpi](https://github.com/yknxh/rlt-openpi)

选择依据：

- 使用 OpenPI/$\pi_{0.5}$ 作为基础 VLA，与本项目完全对齐
- 模块化设计：`--env-factory` / `--intervention-factory` 可插拔，天然支持仿真环境接入
- 强化学习训练循环完整：阶段一 + 阶段二 + 评估脚本齐全
- 70 次提交，代码成熟度高

**辅助参考**：[RL-Token-SmolVLA](https://github.com/RajatDandekar/RL-Token-SmolVLA)

- 展示了如何嵌入 LeRobot v0.4.4 的策略工厂
- 如最终选择在 LeRobot 框架内运行，可借鉴其集成模式

**监督微调基线**：LeRobot 的 `lerobot-eval` + `lerobot/pi05_libero` 检查点

### 5.2 开发路线图

#### 阶段一：监督微调基线验证 + 吞吐率基准线（约 1 周）

**任务**：

1. 配置 LIBERO 仿真环境，确认 gym 接口可用
2. 用 `lerobot-eval` 跑 `pi05_libero` 在 LIBERO-Spatial / Object / Goal / Libero-10 上的评估
3. **关键**：不仅记录成功率，同时记录每个回合的步数，计算平均完成步数和吞吐率基准线

**验证标准**：

- 成功率复现 LeRobot 官方数字（Spatial 97%、Object 99%、Goal 98%、Libero-10 96%）
- 获得完整的步数分布数据

**产出**：监督微调基线数据表 + 吞吐率基准线

#### 阶段二：RL Token 仿真适配（约 2-3 周）

**2a — RL Token 表征训练（离线）**

任务：

1. 从 `pi05_libero` 检查点提取 VLM 词元嵌入接口
2. 参照 rlt-openpi 的 `scripts/train_rl_token.py` 实现编码器-解码器训练
3. 用 LIBERO 演示数据（LeRobot 格式）训练，约 5000 步

验证标准：

- 重建损失 $L_{ro}$ 稳定收敛
- $z_{rl}$ 维度 = 256，编码器/解码器参数量 < 10M

**2b — LIBERO 仿真环境工厂**

任务：

1. 实现 `make_libero_env`，封装 LIBERO gym 环境
    - 观测字典 → 图像 + 本体感知
    - 动作 → `env.step()`
    - 奖励 → `env.check_success()` 稀疏二值奖励
    - 终止 → 成功或达到最大步数
2. 替换 rlt-openpi 的奖励模块（去掉键盘输入，改为自动成功判定）
3. 去掉人类介入工厂（设为 None）
4. 用监督微调演示数据预填经验回放缓冲区作为预热

验证标准：

- 环境能正常重置/推进/渲染
- 预热阶段能正常收集回合到经验回放缓冲区

**2c — 在线强化学习训练 + 评估**

任务：

1. 冻结 $\pi_{0.5}$ + 编码器，启动 TD3 演员-评论家训练
2. 在 LIBERO-Spatial（先选一个任务）上跑在线强化学习
3. 记录训练过程中的成功率和平均步数收敛曲线
4. 扩展到多个 LIBERO 子集做完整对比

验证标准：

- 训练曲线稳定收敛（无策略崩溃）
- 平均完成步数相对监督微调基线有可观测的下降
- 成功率不显著降低

**理智检查（穿插在 2b-2c 之间）**：

- 先用 LIBERO 的真实状态向量（低维坐标）替代 $z_{rl}$ 作为 TD3 输入
- 如果真实状态 + TD3 在 20k 步内无明显改善 → TD3 核心代码有 Bug，必须先修复
- 通过后再切换为 $z_{rl}$ 输入
