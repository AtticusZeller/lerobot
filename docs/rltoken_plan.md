# 实验与研究设计报告 (V3)

**项目名称**：基于 RL Token 的 Flow-VLA 残差强化学习微调——从真机到仿真的验证框架

**英文拟定**：Residual RL Fine-Tuning for Flow-based VLAs via RL Token: A Simulation Validation Framework

---

## 一、研究动机与核心定位

### 1.1 现有技术痛点

以 $\pi_{0.5}$/$\pi_{0.6}$ 为代表的基于流匹配的 VLA 经过大规模预训练和监督微调后，在 LIBERO 等标准任务上已达到 97%+ 成功率。然而高成功率掩盖了两个关键问题：

- **执行效率低下**：VLA 在接触关键阶段频繁出现"探测行为"——反复接近-回退-重调整，导致任务完成步数远超必要。RL Token 原论文在 Ethernet 任务上观测到 VLA 完成步数为强化学习策略的 2-3 倍。
- **全量强化学习微调不可行**：$\pi_{RL}$/RECAP 等方案需解冻 VLA 主干进行同策略训练（PPO/GRPO），样本效率极低（百万级交互），且存在预训练特征灾难性遗忘风险。

RL Token 方法在真机部署中展示了显著的吞吐率提升（关键阶段 3 倍加速），但仅在 $\pi_{0.6}$（闭源）上验证，且强依赖人类接管和人工稀疏奖励，未验证纯仿真自动化训练的可行性。

### 1.2 本研究的核心定位

本项目将 RL Token 方法从真机精密操作迁移至**仿真通用任务环境**，回答以下核心问题：

> **在监督微调基线已经很强（约 97%）的常见操作任务上，RL Token 的残差编辑能否显著提升任务执行效率（吞吐率），同时维持高成功率？**

核心实验设计：冻结 $\pi_{0.5}$ 作为主干网络，基于 rlt-openpi 代码框架实现 RL Token + TD3 演员-评论家网络，在 LIBERO 基准测试上对比**吞吐率**和**成功率**。

## 二、技术架构

### 2.1 系统总览

```
阶段一（离线，RL Token 表征训练）：          阶段二（在线，仿真强化学习训练）：

  π₀.₅（冻结）                               π₀.₅（冻结）
       |                                         |
  VLM 最后一层词元嵌入 z_{1:M}              VLM 词元嵌入 → z_rl
  （去除语言嵌入，仅保留视觉嵌入）               |
       |                                   [RL Token 编码器]（冻结）
  [RL Token 编码器] → z_rl                       |
       |                                   z_rl + 本体感知 + 参考动作块
  [RL Token 解码器] → L_ro                       |
                                            [TD3 演员网络] → 精修动作块
                                            [双 Q 评论家网络] → Q 值
                                                  |
                                            LIBERO env.step() + env.check_success()
```

### 2.2 基础 VLA：$\pi_{0.5}$

- $\pi_{0.6}$ 是 $\pi_{0.5}$ 的直接衍生（同架构，仅预训练数据量、VLM 主干、动作专家规模不同），RL Token 的编码器-解码器接口完全兼容
- LeRobot 提供 `lerobot/pi05_libero_finetuned`（在 LIBERO 上追加 6k steps 监督微调后的检查点），直接作为冻结主干
- VLA 完全冻结（$\alpha = 0$），不做联合微调。`pi05_libero_finetuned` 已在 LIBERO 上充分微调，再做联合微调边际收益极小且有过拟合风险

### 2.3 RL Token 表征提取（阶段一）

#### 2.3.1 核心设计决策

**语言嵌入处理：去除（严格对齐原论文）**

原论文脚注明确说明：

> "In our experiments each task has a fixed language instruction, so we drop language embeddings in this step."

本项目采用 per-task 训练粒度（见 2.3.2），每个任务语言指令固定，因此严格遵循原论文，从 $z_{1:M}$ 中去除语言相关的嵌入，仅保留视觉嵌入送入编码器。

*消融实验（阶段三）：保留语言嵌入的对照组，验证去除/保留对性能的影响。*

**训练粒度：Per-task（每个任务独立训练一个编码器-解码器）**

理由：
- 严格对齐原论文的 "task-specific" 设计
- 编码器-解码器参数量极小（< 10M），训练成本极低（每个任务几分钟）
- 避免小网络跨任务泛化失败的风险——RL Token 的本质是保真压缩，不是泛化预测
- 4 个 Suite × 10 个任务 = 40 个编码器-解码器，串行训练总计不到 2 小时

*消融实验（阶段三）：per-suite 共享编码器-解码器的对照组，验证跨任务泛化能力。*

#### 2.3.2 训练细节

**训练数据**：每个任务全部 50 条演示轨迹（LIBERO 标准配置）。50 条 × 约 200 步/条 ≈ 10,000 个转移样本，对 < 10M 参数的小网络绰绰有余。

**不做 train/val 拆分**：RL Token 的目标是对特定任务的 VLM 嵌入做保真压缩，不是学习泛化映射。训练数据和在线强化学习阶段的数据来自同一任务、同一冻结 VLA，VLM 嵌入分布在同一流形上。$L_{ro}$ 在训练集上稳定收敛即视为训练完成。

**数学形式**：

编码器将 VLM 最后一层视觉嵌入 $z_{1:M}^{vis}$（去除语言嵌入后）压缩为 $z_{rl}$（2048 维）：

$$z_{rl} = g_{\phi}\left(\left[z_{1:M}^{vis}, e_{rl}\right]\right)_{M+1}$$

解码器自回归重建原始嵌入，损失函数：

$$L_{ro} = \mathbb{E}_{\mathcal{D}}\left[\sum_{i=1}^{M} \left\| h_{\phi}\left(d_{\phi}\left([z_{rl}, \bar{z}_{1:i-1}]\right)\right)_i - \bar{z}_i \right\|^2 \right]$$

其中 $\bar{z}_i = \text{sg}(z_i)$（停止梯度），VLA 参数完全不更新。

**质量评估标准**：$L_{ro}$ 稳定收敛即可进入阶段二。仅在阶段二出现问题（Critic 不收敛、策略不改善）时才回头做更细致的表征诊断（t-SNE 可视化、线性探测等）。

### 2.4 在线强化学习：块级 TD3（阶段二）

冻结 $\pi_{0.5}$ + 编码器，训练轻量演员-评论家网络。

**演员网络**（约 170K 参数，2 层 MLP，隐层 256）：

- 输入：$z_{rl}$ + 本体感知状态 + 参考动作块 $\tilde{a}_{t:t+C-1}$
- 输出：精修动作块 $a_{t:t+C-1}$（残差形式，末层零初始化）
- 参考动作丢弃：训练时以 50% 概率将 $\tilde{a}$ 置零，强制演员网络保持独立动作生成能力

**评论家网络**（双 Q 网络，TD3）：

- 宏动作贝尔曼更新：$y = \sum_{i=0}^{C-1} \gamma^i r_{t+i} + \gamma^C \min_{j=1,2} Q_{\theta_j'}(s_{t+C}, a')$
- 双 Q 取最小值减少过估计偏差
- 延迟更新：每 2 次 Critic 更新执行 1 次 Actor 更新

**行为克隆正则化**：

$$L_{actor} = -Q(s, a) + \beta \cdot \| a - \tilde{a} \|_2^2$$

$\beta = 0.5$，约束演员网络不偏离 VLA 先验太远。

**关键超参数**：

| 参数 | 值 | 说明 |
|------|-----|------|
| 动作块长度 $C$ | 5-10 | 宏动作步长 |
| $z_{rl}$ 维度 | 2048 | 信息瓶颈维度 |
| 演员网络 | 2 层 MLP，隐层 256 | LIBERO 任务复杂度对应此规模 |
| 评论家集成 | 双 Q 网络 | TD3 标准配置 |
| 行为克隆系数 $\beta$ | 0.5 | VLA 先验约束强度 |
| 参考动作丢弃率 | 50% | 探索与利用平衡 |
| 更新-数据比 $G$ | 5 | 每个环境步的梯度更新次数 |
| 折扣因子 $\gamma$ | 0.99 | 长期回报折扣 |
| 目标网络软更新系数 $\tau$ | 0.005 | Polyak 平均 |

### 2.5 真机 → 仿真适配

| 模块 | 真机版本（rlt-openpi） | 仿真版本（本项目） |
|------|----------------------|-----------------|
| 环境接口 | `make_franka_env`（Franka+DROID） | `make_libero_env`（LIBERO gym 封装） |
| 奖励信号 | 人类键盘输入（s/f/p） | `env.check_success()` 自动判定，稀疏二值奖励 |
| 人类介入 | VR 控制器接管 | 去掉；用演示数据预填经验回放缓冲区替代 |
| 环境重置 | 人工重置 | `env.reset()` 自动重置 |

## 三、评估指标

### 3.1 主要指标：吞吐率

- **平均完成步数**：成功回合从开始到 `check_success()=True` 的平均步数
- **吞吐率**：$\text{Throughput} = \frac{\text{成功回合数}}{\text{总步数预算}} \times \text{归一化常数}$
- **步数分布**：成功回合步数的中位数、P25、P75

### 3.2 辅助指标：成功率

- 成功回合数 / 总回合数
- 用于确认 RL Token 没有破坏 VLA 原有能力

### 3.3 全部实验均同时报告两个指标

包括阶段一的监督微调基线评估。

## 五、代码实现路线

### 5.1 代码基座

**主要参考**：[rlt-openpi](https://github.com/yknxh/rlt-openpi)（OpenPI/$\pi_{0.5}$，模块化设计，env-factory 可插拔）

**辅助参考**：[RL-Token-SmolVLA](https://github.com/RajatDandekar/RL-Token-SmolVLA)（LeRobot 集成模式参考）

**监督微调基线**：LeRobot `lerobot-eval` + `lerobot/pi05_libero_finetuned`

### 5.2 渐进式开发路线

代码层面一次性搭建完整 pipeline，运行层面从小到大逐步扩展验证。

#### 阶段一：监督微调基线 + RL Token 训练 + 在线强化学习

**1a — 单任务端到端验证（LIBERO-Spatial 第 1 个任务）**

这是最小可行性验证，所有后续工作建立在此基础上。

任务清单：
1. 配置 LIBERO 仿真环境，用 `lerobot-eval` 跑 `pi05_libero_finetuned` 在该任务上的评估
2. 记录成功率 + 每回合步数（吞吐率基准线）
3. 冻结 `pi05_libero_finetuned`，用该任务 50 条 demo 训练 1 个 RL Token 编码器-解码器
4. 监控 $L_{ro}$ 收敛
5. **理智检查**：用 LIBERO 真实状态向量替代 $z_{rl}$ 跑 TD3，确认强化学习核心代码无 Bug
6. 切换为 $z_{rl}$ 输入，跑完整在线强化学习
7. 对比该任务的 SFT-only vs SFT+RLT 成功率和步数

验证标准：
- $L_{ro}$ 稳定收敛
- 理智检查中 TD3 在 20k 步内有明显改善
- 完整流程中成功率不显著降低，平均步数有可观测下降

**1b — 扩展到 LIBERO-Spatial 全部 10 个任务**

- 各训 1 个 RL Token（串行约 1 小时）
- 各跑在线强化学习
- 汇总 LIBERO-Spatial 整体结果

**1c — 扩展到全部 4 个 Suite**

- Object、Goal、Long 各 10 个任务，重复 1b 流程
- 产出完整的表 1 数据
- 此时已有足够数据更新简历

#### 阶段二：消融实验

在阶段一数据基础上进行：

- 去除 RL Token / 去除动作块 / 去除行为克隆正则 / 去除参考动作透传（表 2 前 5 行）
- **语言嵌入消融**：重训 RL Token 时保留语言嵌入，对比 per-task 性能差异
- **训练粒度消融**：per-suite 共享 RL Token 对比 per-task
- 以上均在 LIBERO-Spatial 上进行，避免全 Suite 重复的计算开销

#### 阶段三：扩展实验 + 论文准备

- 少样本设定（表 3）
- 跨 VLA 泛化（$\pi_{0.5}$ → X-VLA）
- 对照组补充（PLD、DSRL、DAgger、HIL-SERL）
- 文献搜索与创新点定位
